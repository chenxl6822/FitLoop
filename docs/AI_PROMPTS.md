# FitLoop 静态图片生成文档

> 目的：把本文件提交给 GPT 图像生成模型，生成 FitLoop 项目所需的 App 图标、启动页 Logo、引导页插图、README 展示图、默认头像、运动图标集和空状态图。
>
> 项目定位：FitLoop 是一个面向高校学生的运动打卡与健康管理 App，核心功能包括 GPS 打卡、传感器计步、拍照打卡、手动打卡、健康数据统计、目标提醒、好友排行和异常申诉。

## 1. 总美术方向

请以“专业移动端产品视觉设计稿”的标准生成，不要做成廉价 AI 插画。整体应像由资深 App 美术和品牌设计师统一绘制，具备清晰的品牌系统、稳定的构图、克制的细节和真实的产品可用性。

Figma 设计复核结论：

- FitLoop 当前代码使用 `#1F8A70` 作为 Material seed color，因此图片资产必须以健康绿为品牌锚点，而不是只使用蓝紫渐变。
- 蓝紫渐变适合做 GPS 轨迹、数据曲线、运动光带、徽章高光，不适合作为全部画面的主背景。
- App 内首批落地图片应优先服务启动页、引导页和默认头像，因为这三处最影响第一印象，也最容易在 Flutter 中稳定调用。
- 图片应给 UI 留出呼吸空间：引导页插图主体位于上半部，底部留给标题、说明文案和按钮；README 首图左侧留白便于排版。

视觉关键词：

- 高校运动健康、可信赖、年轻、轻量、数据感、持续激励。
- 现代运动 App，不要健身房硬核肌肉风，也不要儿童卡通风。
- 像人工制图：几何关系准确、边缘干净、留白舒服、图标笔画统一、光影有逻辑。
- 适合 Flutter 移动端使用：干净、可裁切、可缩放，小尺寸下仍可识别。

品牌色建议：

| 用途 | 色值 | 说明 |
|---|---:|---|
| 主品牌绿 | `#1F8A70` | 当前项目代码的主题色，表达健康、校园、可靠 |
| 活力蓝 | `#4A90D9` | 运动轨迹、科技数据、GPS 感 |
| 渐变紫 | `#7C5CFC` | 智能分析、年轻感、视觉记忆点 |
| 活力橙 | `#FFB057` | 进度、奖励、激励 |
| 背景浅色 | `#F7FAF8` | App 插图背景 |
| 深色界面 | `#101827` | README 产品展示图里的深色 UI |

统一风格：

- Logo 和图标：几何、简洁、轻微 3D 质感，像真实品牌资产。
- 引导页和空状态：精致扁平矢量插画，带柔和体积阴影，不要过度卡通。
- README 首图：真实产品摄影质感，手机屏幕里的 UI 要像真正的 FitLoop App。
- 运动图标：白色线性图标，统一线宽，统一圆角，放在渐变徽章中。

## 2. 通用生成规则

把下面这段“全局指令”放在每个单张提示词前面：

```text
为 FitLoop 运动打卡与健康管理 App 生成静态图片资产。请按专业移动端品牌设计标准制作，画面精致、统一、克制，像人工设计师绘制，而不是随机 AI 插画。保持几何结构准确、边缘干净、留白充足、光影一致。品牌主色为健康绿 #1F8A70，辅助色为活力蓝 #4A90D9、渐变紫 #7C5CFC、活力橙 #FFB057。不要出现水印、乱码、错误文字、英文假字、品牌商标、Nike/Keep/悦跑圈等真实品牌标识。除非提示词明确要求文字，否则图片中不要出现任何文字。
```

通用负面约束：

```text
避免：廉价卡通感、随机装饰、复杂脏背景、过多渐变光斑、畸形手指、错误人体比例、伪文字、乱码、真实品牌 Logo、水印、过曝、低清晰度、模糊边缘、太多小元素、阴影方向混乱。
```

建议生成方式：

- 每次只生成一张图，便于控制质量。
- App 图标、Splash Logo、默认头像使用 1:1。
- 引导页和空状态使用 3:4，后续可裁切到移动端竖屏。
- README 展示图和运动图标集使用 16:9。
- 若生成模型支持质量选项，请使用最高质量或高细节模式。
