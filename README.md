# FitLoop

FitLoop is a campus-focused sports check-in and health management project.

The repository contains only code, tests, deploy configuration, and lightweight engineering notes. Course source documents, generated reports, presentation files, and acceptance artifacts are intentionally kept locally and ignored by Git.

## Project Layout

- `backend/` - Spring Boot modular monolith REST API.
- `mobile/` - Flutter mobile app source.
- `deploy/` - Docker Compose and Nginx deployment assets.
- `.github/workflows/` - CI checks for backend and mobile projects.

## Milestones

1. Repository baseline and CI.
2. Authentication and mobile shell.
3. Sport check-in session flow.
4. Targets, health data, and statistics.
5. Reminders and campus incentive features.
6. Server deployment configuration.

## Git Rule

Commit after each runnable and testable increment. Do not commit `.docx`, reports, or presentation deliverables.
